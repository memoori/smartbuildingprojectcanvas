## Purpose

This section introduces the building owner/operator, describes the building and project scope, states the desired business outcomes, and sets the tone for the entire RFI. It tells vendors: who we are, what we have, and what we're trying to achieve.

## Structure

### 1.1 Issuing Organization
- Organization name and description
- Contact information for RFI inquiries
- RFI timeline (issue date, questions deadline, response deadline)

### 1.2 Building Description
- Building name and address
- Building type (office, hospital, mixed-use, etc.)
- Gross square footage / square meters
- Number of floors
- Year built / last major renovation
- Current occupancy (tenants, employees, visitors)
- Operating hours

### 1.3 Existing Building Systems
- Current BMS/BAS vendor and age
- HVAC system type (VAV, FCU, chilled water, DX, etc.)
- Lighting control system
- Access control system
- Fire/life safety system
- Network infrastructure
- Any other relevant building technology

### 1.4 Project Scope & Desired Outcomes
- Business outcomes this project must achieve (framed as outcomes, not technology)
- Specific use cases being addressed
- Whether this is new construction, major renovation, or technology overlay on existing building
- Project timeline and budget range (if disclosed)

### 1.5 RFI Intent
- Statement that this is a Request for Information, not a commitment to purchase
- Whether this RFI will be followed by an RFP
- How responses will be used (evaluation, shortlisting, etc.)
- Confidentiality provisions

## Guidance for AI Generation

When generating this section:
- Frame all outcomes in business language (NOI, tenant experience, compliance) not technology language
- Include enough building detail for vendors to assess applicability of their technology
- Be transparent about existing systems — vendors need to understand the starting point
- State clearly whether the owner is open to multiple approaches or has specific requirements
- Include the "three rings" baseline where possible: what the owner believes, what records say, and what's operationally true
