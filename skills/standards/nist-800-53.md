## Overview

Building cybersecurity has become critical as IT/OT convergence expands the attack surface of commercial buildings. Smart buildings connect traditionally isolated operational technology (OT) systems — HVAC, access control, elevators, fire panels — to IP networks and cloud platforms, creating pathways for cyber attacks that can affect physical building operations.

The cybersecurity challenge in buildings is unique because IT and OT have fundamentally different priorities:
- **IT:** Confidentiality → Integrity → Availability
- **OT:** Availability → Integrity → Confidentiality

An OT system failure doesn't just cause data loss — it can make a building unsafe, uncomfortable, or inoperable.

## Key Standards

### NIST SP 800-53
The most comprehensive cybersecurity control framework, originally for US federal systems but widely adopted in private sector. Provides a catalog of security controls organized into families (Access Control, Audit, Risk Assessment, etc.). Relevant for building cybersecurity policies, vendor evaluation, and contractual requirements.

### IEC 62443
The international standard specifically for industrial automation and control system security, directly applicable to building automation:
- **IEC 62443-3-3:** System security requirements and security levels (SL 1–4)
- **IEC 62443-4-1:** Product development lifecycle requirements
- **IEC 62443-4-2:** Component security requirements

### ISO 27001
Information security management system standard. Provides a framework for managing cybersecurity across an organization. Relevant for evaluating vendor security practices.

### FIPS 140-3
Cryptographic module validation standard. Relevant for evaluating the encryption implementations in building technology products.

## Why It Matters for RFIs

Cybersecurity is consistently the most overlooked area in building technology procurement. Key concerns:

- **IT/OT convergence risk** — Connecting BAS to the corporate network creates lateral movement opportunities for attackers. Network segmentation is essential.
- **Legacy device vulnerability** — Many BAS controllers run outdated firmware that cannot be patched. OT network scanning can "brick" legacy devices — passive monitoring is preferred.
- **Third-party access** — Vendors, contractors, and service providers often have remote access to building systems with insufficient access controls.
- **Supply chain risk** — Building technology vendors may not have mature security practices. Evaluating vendor security posture is critical.
- **Cyber security often falls between IT and facilities teams** — Neither team fully owns building cybersecurity, creating gaps.

## RFI Questions Related to Cybersecurity

### Vendor Security Posture
- What cybersecurity certifications does your organization hold (SOC 2, ISO 27001, IEC 62443-4-1)?
- Describe your secure software development lifecycle. How do you manage vulnerabilities?
- What is your vulnerability disclosure and patch release process? Provide your patch history for the last 24 months.
- Have you undergone third-party penetration testing? Can you share results (redacted)?

### Product Security
- How does your product implement network segmentation between IT and OT networks?
- What authentication mechanisms does your system support (multi-factor, certificate-based, SAML/SSO)?
- How are device credentials managed? Do any devices ship with default passwords?
- Does your product support encrypted communications (TLS 1.3, BACnet/SC)?
- What logging and audit trail capabilities does your system provide?

### Operational Security
- How do you handle remote access for support and maintenance? What authentication and authorization is required?
- What is your breach notification process and timeline?
- How do you manage firmware updates and security patches across deployed devices?
- Can the building owner restrict or revoke vendor remote access at any time?

### Incident Response
- What incident response support do you provide if a building system is compromised?
- Do you maintain a Security Operations Center (SOC) for your building technology products?
- How do you coordinate with building owners during a cybersecurity incident?
